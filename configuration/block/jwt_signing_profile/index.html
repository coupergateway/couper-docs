JWT Signing Profile

The jwt_signing_profile block lets you configure a JSON Web Token signing profile for your gateway. It is referenced in the jwt_sign() function by its required label.

Block nameContextLabelNested block(s)
jwt_signing_profileDefinitions Blockrequired-

Attributes

AttributesTypeDefaultDescription
claims
object
-

claims for the JWT payload, claim values are evaluated per request

headers
object
-

additional header fields for the JWT, alg and typ cannot be set

key
string
-

private key (in PEM format) for RS* and ES* variants or the secret for HS* algorithms

key_file
string
-

optional file reference instead of key usage

signature_algorithm
string
-

algorithm used for signing: "RS256", "RS384", "RS512", "HS256", "HS384", "HS512", "ES256", "ES384", "ES512"

ttl
string
-

The token's time-to-live, creates the exp claim

Duration

Values of type duration are provided as number string followed by a unit listed below.

Example: timeout = "300s"

Duration unitsDescription
nsnanoseconds
us (or µs)microseconds
msmilliseconds
sseconds
mminutes
hhours

Example

jwt_signing_profile "myjwt" {  signature_algorithm = "RS256"  key_file = "priv_key.pem"  ttl = "600s"  claims = {    iss = "MyAS"    iat = unixtime()  }  headers = {    kid = "my-jwk-id"  }}

A detailed example can be found here.
